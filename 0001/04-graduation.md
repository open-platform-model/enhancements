# Graduation Criteria — `#Platform` Redesign Umbrella

The umbrella spans CUE schema (core), Go kernel (library), catalog repackaging (catalog), and publish-flow tooling (modules). The two graduation transitions below cover the design-side and implementation-side gates respectively. Implementation milestones land in `config.yaml.history` as each piece ships.

## draft → accepted

The umbrella is ready to be sliced for implementation when:

- `01-problem.md` and `02-design.md` are final and reviewed — Goals, Non-Goals, and the three High-Level Approach sub-sections (registry/materialize, catalog/FQNs, `#ctx` channel) are locked.
- Every Open Question in `03-decisions.md` has resolved (each entry carries `resolved-by-D##`, `deferred-to-NNNN`, or `answered`). Every decision (D1..DN) is recorded with Decision / Alternatives considered / Rationale / Source.
- `schemas/target.cue` compiles (`cue vet` from `enhancements/0001/schemas/` passes) and captures the target shape end-to-end: `#Platform` / `#Subscription` / `#SubscriptionFilter`; `#Module` (without `#defines`; carries inline `#ctx { release, components, ... }`); `#Component` (with `metadata.resourceName` override + `#names` + hidden `#release` slot); `#ReleaseIdentity` / `#ComponentNames`; the SemVer `#FQNType`. (`#ModuleContext` / `#RuntimeContext` / `#ContextBuilder` are NOT introduced — see D1.)
- `config.yaml` carries `semver: minor` (already set — `core` is pre-1.0, so even this umbrella's breaking changes ride a minor bump within `@v0`; D12), `area: core` (already set), `affects: [core, library, catalog, modules]` (already set), and `related` reflects whatever follow-up enhancements have been numbered (currently empty; expected to remain empty until 0002+ exist).
- No `{Capitalised}` placeholder strings remain anywhere in `enhancements/0001/` outside HTML comments.
- Cross-References table in `README.md` lists every file path the implementation will touch — verified by checking each path exists today.
- Risks (`05-risks.md`) and Operational Concerns (`06-operational.md`) have concrete content, not placeholders. Alternatives section in `05-risks.md` names the high-level paths not taken (e.g. MAJOR-only with predicate-version; kernel-injected `Catalog.Version`; Module-valued registry kept with per-platform version).
- The core editing protocol's `SPEC.md` plan is sketched — i.e. the list of SPEC sections that will be added/updated when implementation lands is captured in this file or in `02-design.md` `## Integration Points`.

## accepted → implemented

The umbrella is shipped when:

- **Core schema lands** in `opmodel.dev/core@v0` as a minor bump (D12; `core` is pre-1.0 — breaking changes ride minor bumps within `@v0`): `#FQNType` regex updated to SemVer 2.0 (D5); `#MajorVersionType` removed from primitive metadata; `metadata.version` on `#Resource` / `#Trait` / `#Blueprint` / `#ComponentTransformer` is `#VersionType`; `#Module.#defines` removed; `#Module.#ctx { release, components, ... }` added inline; `#Component.#release` hidden injection slot + `#names` block added; `#Component.metadata.resourceName: *name | #NameType` cascade added; `#ReleaseIdentity` + `#ComponentNames` published in new `core/module_context.cue`; `#Platform.#registry` reshaped to `#Subscription` (D7-style root `Catalog` constant convention required of catalogs); `#knownResources` / `#knownTraits` removed; `#composedTransformers` / `#matchers` downgraded to optional kernel-filled slots. (`#ModuleContext` / `#RuntimeContext` / `#ContextBuilder` deliberately NOT introduced — D1.) `core/SPEC.md` co-updated per the core editing protocol with the `#names` lexical-scope authoring caveat from 02-design.md; `core/INDEX.md` regenerated via `task generate:index`. `task check` from the core repo passes.
- **Library kernel rewired**: `Kernel.Registry` field added with default `"ghcr.io/open-platform-model"`; `Kernel.CoreVersion` field added with default `"v0"` (D12 + D24); `library/opm/materialize/` package implemented with the Go-side range parsing (Masterminds/semver v3 per D11), OCI pull + top-level package scan + FQN indexing flow; `Match` signature changed to take `*MaterializedPlatform`; `library/opm/compile/match.go` rewritten with the lookup → always-unify → predicate algorithm (D6) and structured `MissingFQN` (shape `{release, component, fqn, alternatives}` per 06-operational.md) / `UnifyError` / `MaterializeError` diagnostics; CUE's `conflicting values …: file:line file:line` divergence diagnostic surfaced verbatim through the kernel; legacy import `opmodel.dev/core/v1alpha2@v1` removed from every library consumer; library `task check` passes.
- **Library embedded core schema deleted (D24)**: `library/apis/core/` removed entirely (`embed.go`, the embedded CUE files, every test asserting embed boundaries). No `//go:embed` of any core-schema file survives anywhere in the library tree. Kernel loads `opmodel.dev/core@v0` (or whatever `Kernel.CoreVersion` selects) from the OCI registry via `cuelang.org/go/mod` lazily on first `Validate` / `Match` / `Compile` call; reuses the same substrate `Kernel.Materialize` uses for catalogs. `opm/api` and `opm/apiversion` packages deleted (Part B scope; D24 reaffirms). Core-schema pull failures surface through the `MaterializeError` shape with a `kind: "core-schema"` discriminator (06-operational.md). A grep for `//go:embed` against `library/` returns zero hits referencing any `apis/core/*.cue` path.
- **Catalog repackaged** at `library/modules/opm/` (CUE module `opmodel.dev/catalogs/opm@v0`): a new root-package `catalog.cue` file declares `Catalog: #CatalogIdentity` (D7) plus `#Transformers: [#FQNType]: #ComponentTransformer` manifest (D15 — the kernel's sole discovery surface; no package walk); every primitive sources `metadata.version` from `Catalog.Version` and `metadata.modulePath` from `Catalog.ModulePath`; no `#Module.#defines` wrapper anywhere; first SemVer-FQN OCI tag published end-to-end. Module identifier stays on `@v0` per D12.
- **Publish task** in `modules/Taskfile.yml` extended with the temp-build-dir stamping flow per D9: `rsync` source → `.build/catalog/` (excluding `cue.mod/{pkg,gen,usr}`); write `version_override.cue` sibling file setting `Catalog.Version: "<SemVer>"`; `cue vet` from build dir; `cue mod publish` from build dir. CI guard rejects publishes whose `Catalog.Version` resolves to `0.0.0-dev` (D8 hygiene). Rehearsed against a non-production tag at least once.
- **Library fixture migrated**: `library/modules/opm_platform/platform.cue` uses `#Subscription`-shaped registry; imports `opmodel.dev/core@v0`; old fixture deleted, not aliased.
- **`config.yaml.implementation.status = complete`** with `date` set to the landing date of the final piece. `history` carries one event per landed piece (core land, library land, catalog land, fixture migration), each dated.
- `README.md` carries an `> **Implementation status (YYYY-MM-DD).**` quote block whose date matches `implementation.date`.
- `## Deviations from Design` in `README.md` lists every deliberate divergence from the design above (or says "None").
